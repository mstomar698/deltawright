# spec · v0.6 root-cause taxonomy · 2026-07-08

> The **canonical, closed** vocabulary for every root cause Deltawright can diagnose.
> One enum, grounded in real primitive signals, so the six planned diagnosis surfaces
> (pre-flight matcher, side-car, MCP assist, authoring aid, aggregator, engine) speak one
> language and the "single accurate place" cannot drift into three dialects. Ticket #46;
> implemented in `src/host/taxonomy.ts`; governed by **DW-04**.

## Rules

1. **Closed set.** Every diagnosis is exactly one of the 18 codes below, and `unknown` is
   the first-class "unsure" outcome. A diagnosis surface that cannot ground a code MUST
   emit `unknown` — **silence beats a confidently-wrong label** (DW-03).
2. **Grounded.** Each code is justified by ≥1 **real primitive signal** — a concrete thing
   the injected read, the settle result, or Playwright's engine produces — never an
   inferred vibe. `unknown` alone uses the reserved `catch-all` signal.
3. **Verdict is never overridden.** Codes in *verdict-disagreement* annotate the geometry
   ↔ Playwright disagreement (`[geom:]`); Playwright's actionability verdict always wins
   (DW-02). A diagnosis is a hypothesis about *why*, not a competing verdict.
4. **Forward references are intentional.** Some codes are grounded in signals that later
   tickets wire in (`lateStructural` → #49, `stable` → #50). The taxonomy is the contract
   the diagnosis engine (#48) fulfils; a code may exist before its producer does.

## DW-04 — changing the taxonomy

Adding, removing, or renaming a code is a **governed** change, not a casual edit. It
requires, in this order:

1. an **ADR** (`docs/decisions/YYYY-MM.md`) recording the new/changed code and its grounding;
2. a **corpus relabel** (`bench/flake-corpus/`, #51) so ground truth uses the new vocabulary;
3. an **accuracy-harness re-run** (#52) to confirm the CI floors still hold.

The change is mechanically gated by a **frozen lock** in `test/taxonomy.spec.ts`: a
SHA-256 of the sorted code list. Any add/rename flips the hash and fails the test, forcing
the author through the steps above (updating the lock is the deliberate last step). See
`docs/decisions/design-watches.md` (DW-04).

## The codes

Grouped by category. "Signals" are the machine-checkable `PrimitiveSignal`s in
`src/host/taxonomy.ts`; the grounding gloss is how they combine.

### actionability-blocking — Playwright says the target cannot be acted on, and why

| Code | Meaning | Grounding signal |
|---|---|---|
| `covered-by-overlay` | Hit-point occupied by another element; PW + geometry agree it blocks | `hitSelf`=false + `coveredBy` + `playwright-error` |
| `off-screen` | Outside the viewport / scrolled away | `offscreen` / `inViewport`=false |
| `not-visible` | display:none / visibility:hidden / opacity:0 | `playwright-verdict` + `computed-style` |
| `disabled` | Control disabled | `playwright-verdict` |
| `read-only` | Input read-only (fillable-shaped, not editable) | `playwright-verdict` (isEditable=false) |
| `pointer-events-none` | Computed `pointer-events:none` swallows the click | `computed-style` |
| `unstable-animating` | Element mid-animation, not yet stable | `playwright-verdict` + `getAnimations` |

### verdict-disagreement — geometry and Playwright disagree

| Code | Meaning | Grounding signal |
|---|---|---|
| `geom-disagreement` | Geometry vs PW disagree; PW wins, direction surfaced, never overrides | `agreed`=false + `playwright-verdict` |

### membership-attribution — is the reported delta the *right* set?

| Code | Meaning | Grounding signal |
|---|---|---|
| `background-churn` | High background churn may mask the real change | `droppedBackground` |
| `detached-re-render` | Target removed/replaced by a re-render mid-action | `ref-staleified` |
| `settle-timeout` | Settle hit the `maxWaitMs` cap | `hitMaxWait` |
| `suspected-miss-empty` | Zero nodes + cap hit: no-op OR missed effect (unsure) | `empty-delta` + `hitMaxWait` |
| `late-wave-suspected` | Structural mutation after settle resolved (gap-E, #49) | `lateStructural` |
| `stale-rect-suspected` | Post-settle JS reposition moved the target after read (gap-F, #50) | `stable`=false |

### capture-integrity — coverage was incomplete

| Code | Meaning | Grounding signal |
|---|---|---|
| `injection-blocked` | Observer could not be injected (e.g. strict CSP) | `addScriptTag-failure` |
| `cross-boundary-partial` | Cross-origin frame / closed shadow root skipped | `traversal-skip` |

### fallback

| Code | Meaning | Grounding signal |
|---|---|---|
| `pixel-region-fallback` | No DOM delta; screenshot-diff region stood in | `screenshot-fallback` |

### unknown — the first-class "unsure"

| Code | Meaning | Grounding signal |
|---|---|---|
| `unknown` | Insufficient evidence; emitted over a guess | `catch-all` (reserved) |

## Consumers

`import { ROOT_CAUSE_TAXONOMY, ROOT_CAUSE_CODES, type RootCauseCode } from 'deltawright'`.
The diagnosis engine (#48) maps signals → codes; the corpus (#51) labels ground truth in
this vocabulary; the harness (#52) scores per-code. All numbers stay **corpus-relative**
until real-app calibration (#25/#41).
