# spec · v0.1-delta-layer · 2026-07-07

> This spec distills the binding design doc `ideation-agent-delta-layer.md`
> (§3 delta format, §5 architecture, §6 hard parts, §7 scope, §10 go/no-go) into
> the v0.1 milestone contract. The ideation doc remains the source of truth.

## the ask (restated)
Agents driving Playwright get accessibility snapshots that carry no geometry and
no actionability, and must diff full before/after snapshots to learn what an
action did — so they pick elements that "exist" but are covered, off-screen, or
otherwise unclickable, and interactions fail. Build a thin layer that turns ONE
action into a compact, structured delta of *what changed + where it is + whether
it can be acted on*, reusing Playwright's own actionability so the verdict matches
reality.

## approaches considered
- A: Injected page script (MutationObserver + geometry) + host reconciliation against Playwright's actionability engine — reuses proven primitives, alignment by construction.
- B: Reconstruct the delta from raw CDP domains in the host — more control, but reimplements Playwright and risks verdict drift.

## chosen approach
A, because reusing Playwright's `click({trial:true})` for the verdict makes the
delta's "actionable" match what a real action will do, which is the whole point.

## design
`actAndObserve(page, action)` ensures the observer is injected, arms it, runs the
action through Playwright (inheriting auto-wait), waits for settle, drains a
coalesced net set of changed elements, reads geometry + `elementFromPoint` per
node, reconciles each node's actionability with Playwright, and serializes to the
compact §3 text format. Output is a `Delta` (tree of changed nodes with position
and verdict) rendered to tens-to-low-hundreds of tokens.

## what this is NOT
- NOT a browser, a rendering engine, or a Playwright replacement — it is a layer on top.
- NOT a reimplementation of actionability — Playwright is the authority and wins disagreements.
- NOT a general-web tool yet — v0.1 targets DOM-based attribution on a controlled page; no SPA noise tuning, shadow DOM, iframes, canvas fallback, or MCP server.

## hard rules
- MUST NEVER let geometry override Playwright's verdict.
- MUST NEVER drop a mutation-derived node because of geometry (covered/off-screen nodes must survive to carry a NOT-actionable verdict).
- MUST ALWAYS measure and report the delta's token size.
- MUST ALWAYS keep settle detection a simple, labeled, tunable heuristic in v0.1.
