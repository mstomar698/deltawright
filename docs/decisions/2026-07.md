## 2026-07-07 — Build on Playwright and reuse its actionability engine, do not reimplement it

**Decision:** Deltawright stands on Playwright + headless Chromium. The delta's
actionability verdict is obtained from Playwright's own engine via
`locator.click({ trial: true })` (full actionability checks, no click performed).
Our geometry read is an explanatory annotation only; Playwright's verdict is
authoritative and wins any disagreement.

**Context:** The core failure being fixed is "present in the DOM but not
actionable." If we invented a parallel actionability judgment it would drift from
what a real Playwright action does — recreating the very gap we are closing.

**Alternatives rejected:**
- Reconstruct actionability from raw CDP DOM/overlay domains — reimplements Playwright, high drift risk.
- Geometry-only verdict (rect + `elementFromPoint`) — misses stability/enabled/receives-events nuances Playwright already encodes.

**Affected areas:** `src/host/actionability.ts`, `src/host/actAndObserve.ts`.

**Tags:** `architecture` `contract`

## 2026-07-07 — Delta membership is mutation-derived; geometry is annotation, never a filter

**Decision:** The set of reported nodes comes only from coalesced MutationRecords
(net added/removed/attr/text). `getBoundingClientRect` / `getComputedStyle` /
`elementFromPoint` only annotate a node; they never remove it from the delta.

**Context:** Off-screen and covered nodes are exactly the ones we must report (with
a NOT-actionable verdict). Using geometry as a membership filter would silently
drop the two most important cases.

**Alternatives rejected:**
- Filter out non-topmost / off-viewport nodes to "reduce noise" — would drop the covered/off-screen deliverables.

**Affected areas:** `src/injected/observer.ts` (`coalesce`, `readGeometry`, `collect`).

**Tags:** `architecture` `pattern`

## 2026-07-07 — v0.1 settle detection is a deliberately simple, labeled quiescence heuristic

**Decision:** Settle = "at least one mutation seen AND DOM quiet for `quietMs`, OR
`maxWaitMs` elapsed", then await running animations (`getAnimations().finished`,
capped by `animMaxMs`) before reading geometry. Defaults `{120, 2000, 1000}` ms.
The quiet timer never fires before the first mutation (defeats the delayed-insert
trap). This is explicitly a v0.1 heuristic, hardened in v0.5.

**Context:** Settle is the #1 risk (§6). CSS transitions move geometry without
firing mutations, so quiescence alone is insufficient — hence the animation await.

**Alternatives rejected:**
- Fixed sleep — brittle and slow.
- Network-idle only — misses pure-DOM/animated changes.

**Affected areas:** `src/injected/observer.ts` (`waitForSettle`, `settleAnimations`). See DW-01.

**Tags:** `architecture` `performance`

## 2026-07-07 — Author the injected script as TypeScript, bundle with esbuild, inject via addScriptTag

**Decision:** `src/injected/observer.ts` is normal TypeScript (type-checked,
shares types with the host); `src/host/inject.ts` bundles it to a self-contained
IIFE with esbuild at runtime (cached) and installs it with `page.addScriptTag`.

**Context:** Keeps the page script type-safe and unit-testable while still being a
single droppable artifact; `addScriptTag` runs program-level code cleanly,
avoiding `page.evaluate(string)` expression-vs-statement pitfalls.

**Alternatives rejected:**
- Inline a `.toString()`'d function — loses imports/shared types, brittle.
- `page.evaluate(source)` of an IIFE — expression/statement edge cases.

**Affected areas:** `src/injected/observer.ts`, `src/host/inject.ts`.

**Tags:** `tooling`

## 2026-07-07 — v0.1 fixture is served via file://; token size measured with cl100k as a labeled proxy

**Decision:** The controlled fixture loads over `file://` (no static server). Delta
token size is measured with `gpt-tokenizer` (cl100k_base) and labeled a proxy.

**Context:** v0.1 needs a reproducible, dependency-light proof. Exact per-model
tokenization varies; a fixed, offline tokenizer gives a stable, honest number.

**Alternatives rejected:**
- Stand up an HTTP server for the fixture — unnecessary for v0.1.
- Claim model-specific token counts — not reproducible offline.

**Affected areas:** `test/helpers.ts`, `src/host/serialize.ts`, `demo/run-demo.ts`.

**Tags:** `tooling`

## 2026-07-07 — v0.1 actionability verdict is pointer/click-actionability, probed via click trial

**Decision:** The delta's `verdict` answers "can this be clicked?" and is obtained
from a single probe, `locator.click({ trial: true })`, for every node. This matches
the design doc's model (a covered control is NOT-actionable). Role-aware probes
(`fill` editability for text inputs, `selectOption`, `hover`) are deferred to v0.5.

**Context:** An adversarial verification noted `click({trial})` is wrong in both
directions for non-click actions: a covered text input is reported NOT-actionable
though `fill` (no hit-test) would succeed; a readonly input is reported ACTIONABLE
though `fill` throws. For v0.1 we choose one consistent, honestly-labeled semantic
rather than a half-built role dispatch.

**Alternatives rejected:**
- Per-role probe dispatch now — real v0.5 work; premature and under-tested at v0.1.
- Leave "ACTIONABLE" unqualified — misleads for fill/select/hover.

**Affected areas:** `src/host/actionability.ts`, `src/host/types.ts`. See DW-02.

**Tags:** `contract` `architecture`

## 2026-07-08 — #30 attribution stays baseline-authored; the trusted-event anchor is KEEP-ONLY; independent in-window dropping is declined

**Decision:** For #30 (advanced causal attribution) the pre-arm baseline remains the
**sole authority** for dropping background element-insertions (a signature that recurred
≥2× *before* the action). The new trusted-event anchor (`inWindowRecurrence: true`,
default off) is **KEEP-ONLY**: it makes that drop per-root, rescuing the action's own
instance of a background-looking signature when it is DOM-local to, or within
`ANCHOR_RADIUS = 64 px` of, the trusted click. It can never cause a drop and never
promote a signature to background. **Independent in-window dropping** (condemning a
signature on within-window evidence alone, to remove the ~150 ms baseline latency) is
**declined as unsafe** — including the weaker "baseline-count ≥ 1 reinforces" variant.

**Context:** A 4-design adversarial panel (temporal-cadence, locality+recurrence,
signal-fusion, skeptic-minimal) reached a unanimous, adversary-verified result: with only
within-window signal you cannot separate a long causal payload (streamed search results,
a staggered master-detail mount, an SSE generate-stream longer than the settle cap, a
multi-wave refresh) from a recurring background feed — they share signature, cadence,
temporal spread, and a truncated tail, and `maxWaitMs` is a timeout, not a semantic
boundary, so "still arriving at the cap" is information-free. Every independent design had
a high-danger, unclosable **false-drop** (dropping the action's real change — the one
unforgivable error under the "when uncertain, KEEP" invariant). Even baseline-count == 1
corroboration breaks, because `insertSig` keys on `classList[0]` (a shared Tailwind/hashed
class), so count == 1 collisions between an unrelated baseline insert and the action's real
output are common. The anchor rescue, by contrast, only ever *keeps*, survives every panel
scenario trivially (none of those payload signatures are in `bgInsert`), and fixes a latent
false-drop already in the shipped slice (an app reusing a toast class for its own
confirmation).

**Alternatives rejected:**
- Independent in-window drop via cadence/CoV/tail/group-count/far-from-anchor scoring — each adversarially shown to false-drop a real streamed or staggered payload.
- Baseline-count ≥ 1 "reinforcement-only" promotion — breaks on shared-class signature collisions (the CRM-refresh case drops a whole repopulated dashboard).
- Cutting `baselineMs` and recovering the lost drops in-window — the recovery signal is exactly the unsafe one above.

**Affected areas:** `src/injected/observer.ts` (anchor capture, `isActionLocal`, per-root
coalesce), `src/host/actAndObserve.ts` (`inWindowRecurrence`). Tests `test/rescue.spec.ts`.
Full rationale in `docs/summaries/v0.5-inwindow-attribution-decision.md`.

**Tags:** `architecture` `contract`

## 2026-07-08 — Package as a self-contained ESM build; Playwright is a peer, esbuild is not shipped

**Decision:** `npm run build` (tsup + a small `scripts/build.mjs`) emits `dist/`: ESM +
bundled `.d.ts` for the public entry points, executable `deltawright`/`deltawright-mcp`
bins, and a **pre-bundled injected-observer IIFE** (`dist/injected/observer.global.js`) that
the host reads at runtime. `package.json` exports/main/types/bin point at `dist/`; `files`
ships only `dist` + docs; `prepack` builds so publishes are always fresh. `@playwright/test`
is a **peerDependency** (consumers bring their own; the core uses type-only imports),
`gpt-tokenizer` is a runtime **dependency**, and `esbuild` stays a **devDependency**.

**Context:** Before this, `exports` pointed at raw `./src/*.ts` run via tsx, so nothing was
installable and every planned wrapper (reporter/matchers/wait) was un-shippable. Two latent
faults would have made a naive `dist/` install-but-not-run: `inject.ts` esbuild-bundled the
observer from `../injected/observer.ts` **at runtime** (that `.ts` is not in a bundled
`dist/`), and both `esbuild` and `gpt-tokenizer` — real runtime imports — sat in
`devDependencies`.

**Alternatives rejected:**
- Ship `src/` and keep runtime esbuild bundling of the observer — makes esbuild a heavy
  runtime dependency and couples the installed package to a source-tree layout.
- Raw `tsc` per-file `.d.ts` emit — under nodenext a consumer needs `.js` extensions in the
  emitted declarations' relative imports; tsup's bundled, self-contained `.d.ts` sidesteps it.
- Declare subpath exports now for not-yet-built capabilities (`./reporter`, `./matchers`,
  `./wait`, `./aggregate`) — would resolve to missing files and break consumers/typecheck.
  Each capability ticket adds its own subpath entry when it ships. Only `.` and `./mcp` exist.

**Notes:** The `DeltawrightApi` interchange type moved to `src/host/types.ts`; the host
reads `window.__deltawright` via a local cast instead of the injected module's global
augmentation, so the shipped `.d.ts` neither depends on the injected file nor augments a
consumer's global `Window`. Packaging is locked by `test/packaging.spec.ts` (built-package
import under plain node, MCP bin from `dist/`, by-name consumer typecheck under nodenext,
byte-for-byte default-serializer parity src vs. build).

**Affected areas:** `package.json`, `scripts/build.mjs`, `src/host/inject.ts`, `src/cli.ts`,
`src/host/types.ts`, `src/host/actAndObserve.ts`, `src/injected/observer.ts`, `tsconfig.json`,
`.github/workflows/ci.yml`, `test/packaging.spec.ts`.

**Tags:** `packaging` `build` `contract`

## 2026-07-08 — One closed, versioned root-cause taxonomy (DW-04)

**Decision:** All v0.6 diagnosis is expressed in ONE closed enum of 18 root-cause codes
(`src/host/taxonomy.ts`) across six categories, with `unknown` as a first-class "unsure"
outcome. Each code is grounded in ≥1 real `PrimitiveSignal` (a concrete injected-read /
settle-result / Playwright-engine output); `catch-all` is reserved for `unknown`. The set
is versioned and change-gated by **DW-04**: adding/removing/renaming a code requires an ADR,
a corpus relabel (#51), and an accuracy-harness re-run (#52), then updating the frozen SHA
lock in `test/taxonomy.spec.ts` as the deliberate last step.

**Context:** Six planned diagnosis surfaces (pre-flight matcher, side-car, MCP assist,
authoring aid, aggregator, and the engine itself) would otherwise each grow their own
strings for the same causes, so the "single accurate place" fractures into dialects. An
ungrounded or ad-hoc code is exactly how a confidently-wrong label ships — the product's
named anti-goal. Closing and grounding the vocabulary up front, before any diagnosis is
emitted (#48), makes drift a test failure rather than a silent regression.

**Alternatives rejected:**
- Let each surface name causes ad hoc, reconcile later — guarantees drift and a relabel
  churn once the corpus (#51) exists.
- An open/extensible enum — removes the forcing function; a new code could ship without an
  ADR, corpus relabel, or harness re-run, and without a grounding signal.
- Ground codes against live type fields today — the gap flags (`lateStructural` #49,
  `stable` #50) don't exist yet; the taxonomy is the CONTRACT those tickets fulfil, so it
  legitimately names forward signals, validated against a closed `PrimitiveSignal` set.

**Affected areas:** `src/host/taxonomy.ts`, `src/index.ts`,
`docs/specs/v0.6-root-cause-taxonomy.md`, `docs/decisions/design-watches.md` (DW-04),
`test/taxonomy.spec.ts`.

**Tags:** `accuracy` `taxonomy` `contract`
