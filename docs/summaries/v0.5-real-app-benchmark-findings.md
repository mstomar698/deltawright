# summary · v0.5-real-app-benchmark-findings · 2026-07-07

> Issue #23 — first, **directional** read on a real React SPA. Methodology red-teamed
> before running (see `docs/plans/v0.5-real-app-benchmark.md`). **This is not the
> admissible §10 GO/KILL statistic** — that needs a third-party corpus (#25). It is a
> validated harness + honest first signal on a real framework.

## TL;DR
On a real React app, the core Deltawright bet **holds where it should and breaks where
the design predicted** — the benchmark grounds the v0.5 priorities in data rather than
killing or vindicating the idea:

- **(a) token win — REAL vs the realistic incumbent.** Versus re-dumping the full a11y
  tree each step (what Playwright MCP actually does), the delta is dramatically smaller
  and the gap *widens with page size*. Versus a bare before/after diff the full delta is
  larger — but at **information parity** (delta stripped to what-changed-only) it is
  competitive-to-smaller, and it still carries geometry + actionability the diff cannot.
- **(b) noise — CONFIRMED PROBLEM on a live SPA**, exactly as §6.2 warned. Time-window
  attribution sweeps up background re-render churn (noise ratio ≫ 1; null-action probe
  shows a non-zero false-positive floor). The fix is causal attribution (#15). Nuance:
  the incumbent diff is polluted *just as much*, so noise is not unique to Deltawright.
- **(c) settle — CONFIRMED PROBLEM on continuously-updating pages**: the quiescence
  heuristic never goes quiet and always hits the maxWait cap — but it did **not miss**
  the change. Robust settle (#13) is the fix.
- **TIME — a real regression the token score hides**: the delta pays O(nodes) serial
  trial-clicks, so on noisy pages it is multiples slower than snapshot+diff. Async/
  bounded reconciliation (#18) is the fix.

## Setup (measured honestly)
- App: a real React 19 SPA (`bench/apps/react-app.tsx`) — a data table sized by config,
  an optional live-ticker that re-renders every row's Live cell (genuine React
  reconciliation churn), and a modal-open action.
- Three arms per action: **DELTA** (`actAndObserve`), **BEFORE+AFTER+DIFF** (the §10
  incumbent — minimal LCS line-diff of two a11y snapshots), **RE-SNAPSHOT** (full a11y
  tree, the MCP default). Delta and incumbent share the **same settle wait**.
- Info-parity arm: **delta-lite** = the delta stripped of geometry + verdict, so its
  tokens are comparable to the diff.
- **Null-action probe**: arm→settle around a NO-OP on the churning app → every reported
  node is a pure false positive (noise floor).
- Tokens: cl100k proxy (pinned; see the bug below). **Ratios are the tokenizer-robust
  signal.** Medians of 3 trials.

## §10 assessment

**(a) token win — SURVIVES.**
- **At information parity** (delta‑lite vs the a11y diff): **0.87× → 0.15×** — the delta
  encodes the *same* change‑set more compactly than the incumbent diff in every scenario,
  ~6× more compactly under noise. This is the fair, apples‑to‑apples fight, and the delta wins it.
- **Vs the full re‑snapshot** (MCP default): 0.16× (small) → **0.01× on a large page**
  (99 vs 14,524 tokens — 100× smaller), scaling with page size exactly as the thesis predicts.
- The full delta *additionally* carries geometry + a Playwright‑aligned actionability verdict
  the diff/snapshot structurally cannot — a separate capability axis, never folded into the token score.

**(b) noise — CONFIRMED PROBLEM (as §6.2 predicted), with an honest nuance.**
- Quiet page: noise ratio **1.0**, null‑action false positives **0** — the coalescer is exact when nothing else churns.
- Live SPA: noise ratio **13.8–76.3**; the null‑action probe (a NO‑OP action) reports **51 / 301**
  pure‑churn nodes, proving the pollution is genuine background re‑render, not benchmark construction.
  Cause: v0.1's time‑window attribution. Fix: causal attribution (#15) — which would drop the noisy
  delta back toward the modal's ~4 nodes / ~99 tokens.
- Nuance: the incumbent diff is polluted *just as much* (its tokens balloon too), and delta‑lite still
  encodes that noisy change‑set ~6× more compactly than the diff. Noise is a shared change‑detection
  problem, not unique to Deltawright.

**(c) settle — CONFIRMED PROBLEM on continuously‑updating pages.**
- Quiet: quiesces at ~163 ms, never caps, captures the change 3/3.
- 60 ms ticker: never goes quiet → hits the 2 s maxWait cap **every** time (3/3) — but did **not miss**
  the change (captured 3/3). The cost is latency + noise accumulation, not a missed delta. Fix: robust settle (#13).

**TIME — a real regression the token score would hide.**
- The delta's O(nodes) serial trial‑clicks make it **1.9× (small‑noisy) to 5.8× (large‑noisy) slower**
  than snapshot+diff (12.3 s vs 2.1 s on large‑noisy). Reported explicitly so a token win can't mask it.
  Fix: async/bounded reconciliation (#18).

**Overall: not a clean §10 GO, not a KILL.** The core value — delta ≪ re‑dumping the page, more compact
than the incumbent diff at parity, and carrying geometry+actionability — **holds on real React**. The three
v0.5 hard problems (noise, settle, time) materialize exactly as the design doc predicted and are now
**quantified**, converting them from risks into scoped, measured work (#15, #13, #18). The admissible
verdict awaits the third‑party corpus study (#25).

## What this proves — and does NOT
- **Proves (directionally):** the harness is fair (steel-manned incumbent, info-parity,
  time + noise + null-action measured); the token win vs re-snapshot is real and scales;
  the noise/settle/time weaknesses are real and now quantified on genuine React.
- **Does NOT prove:** a §10 GO/KILL. The app is team-authored (author bias), single
  framework, no navigation/reorder interactions, no third-party corpus, N=3 not ≥30,
  cl100k not Claude's tokenizer. The admissible study is **#25**.

## Bug found while doing this
The `gpt-tokenizer` 3.x bump silently changed the default `encode` from cl100k to
o200k, so every "cl100k" label (serialize, README, demo, CHANGELOG) had become wrong.
Fixed by pinning `gpt-tokenizer/encoding/cl100k_base` so the label is true and can't
drift, with a note that it is an OpenAI proxy and ratios are the robust signal.
