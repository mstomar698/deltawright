# summary · v0.1-milestone · 2026-07-07

## go / no-go (against ideation §10)

**Verdict: GO — with caveats. Proceed into v0.5.**

v0.1 clears a **narrower engineering sub-bar**, not §10 as literally written:

- ✅ The `arm → act → settle → collect → reconcile` primitive works end-to-end, with
  **no before/after full-snapshot diff**, green 12/12.
- ✅ On the controlled north-star cases, the pointer-actionability verdict **agrees
  with what a real Playwright action then does** (popup ACTIONABLE incl. the textbox
  via a real `fill`; covered → NOT-actionable; off-screen → NOT-actionable; disabled →
  NOT-actionable with the geometry disagreement surfaced).
- ✅ All five non-negotiable constraints hold (no browser fork; actionability reused
  from Playwright and it wins disagreements; delta token-tiny; DOM-based/controlled
  scope only; settle is a simple, labeled, tunable heuristic).

**Why not a clean GO (stated plainly — these are honest limits, not proven wins):**

- **§10's bar is "on your own real target app."** v0.1 ran only on a synthetic,
  deliberately observer-friendly fixture. **None of the three §10 kill-criteria are
  retired** — all three need real-app data:
  - **(a) token win — NOT shown, and negative here.** Case 1's delta is 101 tokens vs
    a ~90-token full-page a11y snapshot of the same page: the delta is *larger*. The
    real incumbent (snapshot **before + after + diff**) was never measured. The token
    bet rests entirely on unmeasured large-SPA extrapolation.
  - **(b) mutation noise — untested.** Coalescing was validated only against synthetic
    single-shot mutations with no framework. Attribution is **time-window-scoped, not
    causal**: every mutation between `arm()` and settle is attributed to the action.
  - **(c) settle reliability — a labeled heuristic** proven only for a single change
    arriving within `[quietMs, maxWaitMs]` on an otherwise-static page. A change after
    `maxWaitMs` yields an empty delta (now surfaced as a SUSPECTED MISS, not silently);
    a sub-`quietMs` ticker never quiesces and pays the full cap.
- **The trial↔real "match" is internal consistency, not independent ground truth** —
  both are the same Playwright engine. The genuinely independent evidence is the
  **a11y-snapshot-says-interactive vs delta-says-NOT-actionable** contrast (cases 2–3).

Bottom line: the idea is a **buildable, constraint-clean, correct-on-controlled-cases
primitive** worth the v0.5 month — *not* a claim that the idea is validated or that
§10's real-app question is answered.

## what shipped
The v0.1 delta primitive: an injected MutationObserver + coalescer + settle heuristic
+ geometry/`elementFromPoint` reader (`src/injected/observer.ts`), a host wrapper
(`actAndObserve`) that reconciles each changed node's actionability against
Playwright's authoritative `click({trial:true})`, and a compact serializer with a
token count. Four controlled cases (popup / covered / off-screen / disabled), a
12-test suite (5 coalescer units + 4 net-difference units incl. the removal path + 4
north-star cases), a demo, and the full OSS + SDLC scaffolding.

## what did NOT ship (and why)
- Token-savings evidence — out of scope for a controlled page; needs a real app measured against before+after+diff (v0.5).
- Mutation-noise filtering / causal attribution — v0.5 (§6.2); today attribution is temporal.
- Robust settle detection — v0.1 is a deliberately simple heuristic (DW-01).
- Role-aware actionability probes (`fill`/`select`/`hover`) — the verdict is click-only in v0.1; a covered input reports NOT-actionable though `fill` would succeed, a readonly input reports ACTIONABLE though `fill` throws. Labeled honestly; v0.5.
- Shadow DOM / iframe / canvas fallback / MCP server / cross-browser / terminal — later, per §7.

## surprises
- The adversarial verification found a **real correctness bug the tests would have missed**: a net-removal of a *pre-existing* element that is also moved in the same window was silently dropped (the `removed` path had zero test coverage). Fixed via record-order `firstTouch` + new tests.
- The tool's **headline differentiator was invisible**: the geometry↔Playwright disagreement was computed but never serialized, and no `agreed===false` case existed. Fixed: added the disabled-button case and the `[geom:…]` marker.
- The **token framing was backwards** in the first draft ("modest win"): the delta is actually larger than the snapshot on this page. Corrected in README + demo.

## what is next
- v0.5, in §6 priority order: harden settle (DW-01); real-app noise filtering + causal attribution; role-aware probes (DW-02); async/bounded reconciliation (today it's serial O(nodes), each NOT-actionable node pays the full trial timeout); shadow DOM + same-origin iframes; screenshot-diff fallback; MCP packaging.
- The decisive v0.5 experiment: run a real target app, measure the delta against the actual before+after+diff incumbent, and see whether §10 kill-criteria (a)/(b)/(c) survive.
