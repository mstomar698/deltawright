# summary · v0.5-gwt-legacy-findings · 2026-07-08

> Issue #41 — does Deltawright help where Playwright "fails hard" on legacy GWT / 2000s
> portals? Honest answer: **partially, in a narrow niche, and it has its own gaps.**
> **GO** as a better *moment* + stable *handle* + independent *second-opinion*; **NO-GO**
> as "Deltawright fixes Playwright's GWT actionability." Driven by a design panel whose
> adversary demolished the strawman baselines. Demo: `npm run demo:gwt`; tests:
> `test/gwt.spec.ts`; fixture: `test/fixtures/gwt.html`.

## The correction that shaped this

The tempting pitch — "Playwright fails on GWT, Deltawright fixes it" — **does not survive
scrutiny**, and shipping it would collapse on the exact pain point it targets. Idiomatic,
web-first Playwright (`await expect(locator).toBeVisible()`, auto-waiting text/CSS
locators) polls straight through GWT's deferred-command cascade and **passes**. The naive
failures a demo could show — `accessibility.snapshot()` racing a deferred render,
`waitForLoadState('networkidle')` as the wrong signal, manual `boundingBox()` coordinate
math reading a pre-reposition rect, `getByRole()` finding no role on a `<td>` — are
**strawmen**: a competent engineer rewrites them in three idiomatic lines and they go
green. So every baseline here is paired with its idiomatic **steel-man**, and only cases
that genuinely defeat idiomatic Playwright are claimed as wins.

## What GWT actually does (reproduced in `gwt.html`)

- **Deferred commands, no network:** `Scheduler.scheduleFinally` (stack-unwind) →
  `scheduleDeferred` (≈`setTimeout(0)`) → layout frame, so widgets attach *after* the click
  promise resolves, with zero network I/O.
- **Glass overlay:** `PopupPanel.setGlassEnabled(true)` inserts a full-viewport
  `div.gwt-PopupPanelGlass` that intercepts pointer events (`gwt-` theme classes are stable,
  unlike obfuscated CssResource app classes).
- **Self-repositioning `DialogBox`**, delegated event dispatch on ancestors, role-less
  hashed-class widgets, a hidden history iframe.

## Findings — three scoped wins, three honest limits

| Case | What Deltawright does | Honest label |
|---|---|---|
| **A** deferred-observe | A **locator-free, network-free** structural-quiescence settle catches the deferred render and returns a stable `data-dw-ref` handle + final rect. A zero-wait read misses it. | ✅ **Genuine win** — but only when *observing consequences* with no target to name; for a known locator, idiomatic `toBeVisible()` also passes. |
| **B** glass overlay | Surfaces `coveredBy=div.gwt-PopupPanelGlass` as a compact structured pre-action fact; a sub-second labeled `NOT-actionable(covered)` vs a hand-rolled readiness heuristic's 30s click-hang. | ⚠️ **Diagnostic + latency, not a correctness fix** — it *agrees* with Playwright (already correct); DW cannot dismiss the glass. |
| **C** disabled/covered | Makes the geometry↔Playwright disagreement visible (`[geom:ACTIONABLE]`) while Playwright's authoritative verdict still ships. | ✅ **Visibility win.** |
| **D** role-less cell | Reports "structure changed here" with ref + rect, `role:null`/`name:null`. | 🔎 **Identification limit** — invents no semantics GWT omitted; its textContent name is the same handle `getByText` already uses. |
| **E** two-wave render | Inter-wave gap > `quietMs` (120 ms) ⇒ settle resolves on wave 1 and **silently** under-reports wave 2 (`hitMaxWait:false`, no suspected-miss line). | 🔴 **Silent correctness gap.** |
| **F** JS-timer recenter | `getAnimations()` is empty for JS style writes, so `settleAnimations` is a no-op and the annotated rect is stale (verdict stays correct; only the geometry annotation). A CSS transition is awaited correctly. | 🔴 **Geometry-staleness gap.** |

## What Deltawright CANNOT do (stated on the same slide as the wins)

- Cannot dismiss the glass or make a covered/disabled control clickable — those verdicts
  are **correct**, and DW must never rescue them.
- Cannot semantically know GWT deferred/RPC completion — structural quiescence is a
  mutation-*timing* heuristic (hence gap E).
- Cannot bound an RPC wait it has no signal for (only lever: raise `maxWaitMs`).
- Cannot keep the annotated rect correct through a JS-timer recenter (gap F).
- Cannot keep a `data-dw-ref` valid across a GWT re-render/detach — it staleifies exactly
  like any Playwright locator.
- The fixture is **hand-authored synthetic DOM**, not compiled GWT — it proves only that
  "Deltawright behaves correctly on a DOM we assert resembles GWT."

## The checksum

`src/host/checksum.ts` fingerprints the delta's structure + semantics (kind, role/tag,
name, verdict, `[geom]` disagreement, size bucket, coverage) while dropping raw pixels,
refs, styles, timing, and record order — and hashing `gwt-*` classes stably while
tokenizing obfuscated app classes (rebuilt each compile). It is a **regression guard
only**: a matching hash asserts output == captured output. It is *not* evidence of GWT
correctness and can pass on a fictional DOM.

## Go / no-go

- **GO** to position Deltawright on legacy GWT as: *a better completion **moment** for
  observe-what-changed (no locator needed), a stable **handle** on the appeared node, and
  an independent **coverage / disagreement second-opinion**.* All three hold in
  `test/gwt.spec.ts`, and the two silent gaps (E, F) are asserted, not buried.
- **NO-GO** to claim Deltawright "fixes Playwright's GWT actionability," to lean on
  strawman baselines, or to attribute the actionability *verdict* to settle (the
  trial-click's own auto-wait gets the verdict; settle only changes *which* nodes enter
  the delta and the annotated rect).

## The one dependency for full credibility

**Calibrate to a real portal trace** before any framework-specific claim or committed
golden hash that purports to represent a real app: the actual glass class name, the
dispatch model (delegated vs `sinkEvents`), the frame tree (is the app DOM inside the
`xsiframe` sandbox?), whether `aria.client.Roles` is present (which would change case D
entirely), and the real Scheduler primitive + timings. This needs the owner's GWT app —
it pairs naturally with the #25 real-app benchmark. Until then this remains a faithful
*synthetic* reproduction, labeled as such.
