# plan · v0.5-real-app-benchmark · 2026-07-07

> Issue #23 — the decisive §10 experiment. Started as the first v0.5 slice.

## goal
On a real React SPA (not the synthetic v0.1 fixture), measure the Deltawright delta
against the actual incumbent (snapshot before + after + diff, and full re-snapshot)
across page sizes and background-noise levels, and honestly test whether the ideation
§10 kill-criteria survive: (a) token win, (b) mutation noise, (c) settle reliability.

## what this plan is NOT doing
- NOT claiming to test production public websites — it uses a real React framework app
  served locally for reproducibility. External-validity limits are disclosed.
- NOT fixing the weaknesses it finds (causal attribution #15, robust settle #13,
  async reconciliation #18) — it MEASURES them so those issues are grounded in data.
- NOT an LLM-in-the-loop eval — it measures the artifacts (tokens, nodes, timing), not
  downstream task success.

## hard rules
- MUST NEVER straw-man the incumbent: count the DIFF an agent actually reads (minimal
  line-diff), not just a full snapshot; report the full snapshot separately.
- MUST NEVER fold the delta's geometry + actionability into the token score — it is a
  CAPABILITY the a11y snapshot/diff structurally lack, reported as its own axis.
- MUST ALWAYS give the delta and the incumbent the SAME settle wait, and record TIME,
  so a token win cannot hide the delta's O(nodes) trial-click cost.
- MUST ALWAYS report medians over multiple trials (noise makes single runs vary) and
  disclose when the delta fails to capture the intended change.

## scope
- `bench/apps/react-app.tsx` · real React SPA: table (sized by config), live-ticker
  noise (real re-render churn), modal-open action.
- `bench/bundle-app.ts`, `bench/load-app.ts` · esbuild-bundle + inject the app (no network).
- `bench/diff.ts` · LCS line-diff (the incumbent's before+after+diff output).
- `bench/run-bench.ts` · the harness: delta vs re-snapshot vs before+after+diff; tokens,
  noise ratio, settle, timing; medians.
- `docs/summaries/v0.5-real-app-benchmark-findings.md` · the honest §10 verdict.

## implementation paths considered
- A: local real-framework app (React), reproducible + CI-able — chosen.
- B: drive real public sites — higher external validity but flaky, non-reproducible,
     may block automation; deferred to a manual follow-up.

## chosen path
A, because it exercises genuine React reconciliation/noise deterministically, so the
numbers are trustworthy and repeatable; B is noted as a follow-up for external validity.

## slices
- A1 · should_render_real_react_app_with_configurable_size_and_noise — done (smoke-verified).
- A2 · should_measure_delta_vs_resnapshot_vs_before_after_diff — the harness.
- A3 · should_report_noise_ratio_settle_and_timing_per_scenario — metrics.
- A4 · should_write_an_honest_go_no_go_findings_report_mapped_to_ten — the deliverable.

## risks
- Straw-manning the incumbent → mitigated by counting the minimal diff + shared settle (red-team methodology).
- Flattering Deltawright by ignoring time → mitigated by recording delta/incumbent wall time.
- Overclaiming "real app" from a local fixture → mitigated by explicit disclosure; public-site test deferred.
- Benchmark runtime blows up on large-noisy (O(nodes) trial-clicks) → itself a finding; bounded via row caps / trial count.

## admissibility (what this slice IS and IS NOT)
Red-teamed conclusion: a **team-authored** app is **directional evidence + a validated
harness**, NOT the admissible source of a §10 GO/KILL statistic (author bias — we could
unconsciously build an app our coalescer handles well). This slice delivers:
- the harness (delta vs before+after+diff vs re-snapshot; tokens, info-parity, noise,
  null-action floor, settle, time; medians), and
- a first honest, directional read on a real React app.

The **admissible go/no-go** (a separate follow-up slice) requires, per the red-team:
1. Tier-1 corpus of 2–3 THIRD-PARTY, unmodified, popular framework apps (React + Vue
   RealWorld/Conduit, official examples), pinned by commit+lockfile, chosen before any
   delta is inspected.
2. A pre-registered interaction set spanning all four quadrants — incl. a NAVIGATION
   (re-snapshot's home turf) and a keyed-list REORDER (the delta's stress case).
3. N≥30 reps/cell on an isolated runner, warm-up discarded; a hand-labeled ground-truth
   change set for precision/recall (not just a node-count ratio).
4. A structure-aware, order-insensitive diff (the current LCS line-diff over-reports on
   reorders — fine for insert/text/attr scenarios here, disclosed as a limit).
5. Tier-2 production sites captured to HAR/WARC and replayed offline for reproducibility.
