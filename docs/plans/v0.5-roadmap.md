# roadmap · v0.5 · 2026-07-07

## goal
Turn the proven v0.1 primitive into the differentiated core: reliable on real apps,
noise-filtered, and honest about actionability for every action type — then run the
decisive experiment that answers the ideation §10 go/no-go on a real target app.

## priority order (ideation §6, hardest first)
Each item is a tracked GitHub issue labelled `v0.5`.

1. **[#13] Harden settle detection (DW-01)** — DONE. Robust **structural-quiescence**
   settle (resolves on structural quiet, ignoring non-structural background churn) +
   regression suite; benchmark-validated (live-page settle 2000 ms → ~155 ms, no cap,
   capture preserved). **DW-01 retired.** Residuals (attr/text-only effect + element-
   adding churn on live pages) → #15.
2. **[#15] Mutation-noise filtering + causal attribution on real SPAs** — DONE.
   Pre-arm baseline footprint excludes background churn (channel-granular, keeps
   real changes). On the live-SPA benchmark: **noise_ratio 13.8/76.3 → 1.0**,
   null-action false positives **51/301 → 0**, large-noisy delta **8958 → 99 tokens**
   — the delta on a churning page is now identical to a quiet page. This is the
   decisive token win vs the incumbent diff (delta ÷ diff → ~0 on noisy pages).
   Residual: element-adding background churn (toasts/virtualized) → future.
3. **[#17] Role-aware actionability probes** — DONE. Probe per node type (fill for text
   inputs, selectOption for selects, click for the rest) so the verdict matches the
   real action: a covered input is ACTIONABLE (fillable), a read-only input is not,
   disabled reports `disabled`. Verified against real fill/select. DW-02 stays active
   (the invariant is ongoing; #17 strengthened compliance). `hover`-only targets → future.
4. **[#18] Async/bounded actionability reconciliation** — DONE (bounded-concurrent,
   default `reconcileConcurrency: 12`): reconcile ~8× faster on a 305-node delta
   (overall 12.3 s → 3.4 s), verdicts + real-app recall (30/30) preserved. Remaining
   large-noisy wall is the settle cap (#13).
5. **[#19] Shadow DOM + same-origin iframe traversal.**
6. **[#20] Screenshot-diff fallback for canvas / cross-origin regions.**
7. **[#22] MCP server surface** (`act_and_observe`, `describe_changes`, `full_snapshot`).

## the decisive experiment
- **[#23] Real-app token benchmark vs snapshot-before+after+diff.** DIRECTIONAL FIRST
  CUT DONE (`bench/`, `docs/summaries/v0.5-real-app-benchmark-findings.md`): on a real
  React SPA the token win holds (delta 0.01× a large re-snapshot; 0.15–0.87× the diff
  at info-parity) while (b) noise and (c) settle are confirmed problems and the delta's
  O(nodes) time cost is quantified — grounding #13/#15/#18 in data. NOT a §10 GO/KILL
  (team-authored app, N=3).
- **[#25] Admissible go/no-go** — the rigorous version of #23. FIRST CUT DONE
  (`npm run bench:admissible`, `docs/summaries/v0.5-admissible-benchmark-findings.md`):
  real third-party TodoMVC React (pinned), structure-aware diff, a navigation
  interaction, recall 30/30, settle 0/30-capped, and a consistent token win over
  re-snapshot (0.19–0.45×) — the author-bias objection is removed and the core bet holds
  on code we didn't write; the actionability time cost is confirmed (#18). STILL OPEN:
  a second framework (Vue), Tier-2 production sites via HAR replay, and Claude's
  tokenizer as the primary counter. Not "validated" until those land.
