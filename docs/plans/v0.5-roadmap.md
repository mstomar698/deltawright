# roadmap · v0.5 · 2026-07-07

## goal
Turn the proven v0.1 primitive into the differentiated core: reliable on real apps,
noise-filtered, and honest about actionability for every action type — then run the
decisive experiment that answers the ideation §10 go/no-go on a real target app.

## priority order (ideation §6, hardest first)
Each item is a tracked GitHub issue labelled `v0.5`.

1. **[#13] Harden settle detection (DW-01)** — the #1 risk. Robust settle
   (quiescence + auto-wait + optional network-idle), corpus-validated, with a
   suspected-miss signal. Retires design-watch DW-01.
2. **[#15] Mutation-noise filtering + causal attribution on real SPAs** — where the
   value is earned (§6.2). Semantic-attribute whitelist, text net-compare, and a move
   from time-window-scoped to causal attribution.
3. **[#17] Role-aware actionability probes (DW-02)** — probe `fill`/`selectOption`/
   `hover` per node instead of click-for-everything, so the verdict matches the
   specific action. Relaxes/retires design-watch DW-02.
4. **[#18] Async/bounded actionability reconciliation** — today it is serial
   `O(nodes)` and each NOT-actionable node pays the full trial timeout.
5. **[#19] Shadow DOM + same-origin iframe traversal.**
6. **[#20] Screenshot-diff fallback for canvas / cross-origin regions.**
7. **[#22] MCP server surface** (`act_and_observe`, `describe_changes`, `full_snapshot`).

## the decisive experiment
- **[#23] Real-app token benchmark vs snapshot-before+after+diff.** This is the
  experiment ideation §10 says decides go/no-go: on a real target app, measure the
  delta against the *actual* incumbent (before + after + diff) and test whether the
  three kill-criteria — (a) token win, (b) noise, (c) settle — survive. Everything
  above feeds this; nothing about the idea is "validated" until this passes.

## not v0.5 (later, per ideation §7)
- Cross-browser (Firefox/WebKit).
- Terminal observability — render the changed region into a tmux pane (human
  verification, **not** agent input). Comes at v1.0.
- Test-gen opinion: stable-selector candidates + assertion suggestions per node.
