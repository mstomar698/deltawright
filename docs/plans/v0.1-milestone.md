# plan · v0.1-milestone · 2026-07-07

## goal
Prove the Deltawright core primitive on a controlled north-star case: a single
Playwright action (`actAndObserve`) returns a compact, structured delta of what
changed on the page, where it is (geometry), and whether it is actionable — with
an actionability verdict that matches what Playwright actually does, and with NO
before/after full-snapshot diff. Success is the popup click yielding dialog +
geometry + interactive children + accurate verdict in tens-to-low-hundreds of
tokens, plus correct NOT-actionable verdicts for covered and off-screen variants.

## what this plan is NOT doing
- Real-framework (React/Vue/Angular) mutation-noise tuning — deferred to v0.5.
- Shadow DOM / iframe traversal — deferred to v0.5.
- Canvas / WebGL screenshot-diff fallback — deferred to v0.5.
- A full MCP server surface (`act_and_observe`, `describe_changes`, `full_snapshot`) — v0.5.
- Cross-browser (Firefox/WebKit) and terminal rendering — later milestones.
- Robust settle detection — v0.1 ships a deliberately simple, labeled heuristic.

## hard rules
- MUST NEVER build, fork, or embed a browser or rendering engine; the tool stands on Playwright + headless Chromium only.
- MUST NEVER reimplement Playwright's actionability judgment as the source of truth. Geometry is an *annotation*; Playwright's verdict is authoritative and WINS every disagreement.
- MUST ALWAYS derive the reported delta membership from observed mutations only; geometry/`elementFromPoint`/rect are annotations, never membership filters (so covered and off-screen nodes are never dropped before their verdict is computed).
- MUST ALWAYS emit a token-tiny delta (target: low-hundreds of tokens for the popup case) and measure it.

## scope
- `src/injected/observer.ts` · injected page script: one MutationObserver, coalescer, settle heuristic, geometry + `elementFromPoint` read, `data-dw-ref` stamping.
- `src/host/inject.ts` · esbuild-bundle the injected script and install it idempotently.
- `src/host/actAndObserve.ts` · arm → act (via Playwright) → settle → collect → annotate.
- `src/host/actionability.ts` · reconcile geometry vs Playwright trial-click; Playwright wins.
- `src/host/serialize.ts` · compact §3 text format + cl100k token count.
- `src/host/types.ts` · shared interchange + delta types.
- `test/fixtures/northstar.html` · three controlled cases (popup / covered / off-screen).
- `test/*.spec.ts` · coalescer unit tests + north-star validation with verdict-matches-reality.

## implementation paths considered
- A: Injected page-script (MutationObserver + geometry) bundled by esbuild, host reconciles with Playwright trial-click — reuses Playwright's engine, no parallel actionability.
- B: Raw CDP DOM/overlay domains from the host — more control, but reimplements much of what Playwright already exposes and is harder to keep aligned with real actions.

## chosen path
A, because it inherits Playwright's actionability engine directly (via `click({trial:true})`)
so the verdict cannot drift from the real action, and it keeps the observation cost proportional
to the change, not the page.

## slices
- A1 · should_install_injected_observer_and_coalesce_records — one MutationObserver, buffer, `takeRecords`+disconnect on drain.
- A2 · should_report_added_subtree_as_root_plus_interactive_children — subtree-root reduction; live traversal for interactive descendants.
- A3 · should_net_add_then_remove_to_nothing — net-set logic (added∧connected, removed∧disconnected∧¬born).
- A4 · should_read_geometry_and_elementFromPoint_per_changed_node — rect + computed style + center hit-test; animations awaited first.
- A5 · should_reconcile_actionability_with_playwright_trial_click — geometry annotation + authoritative trial verdict; Playwright wins; agreement flagged.
- A6 · should_emit_compact_delta_and_measure_tokens — §3 format, nested by containment, token count reported.
- A7 · should_mark_covered_and_offscreen_NOT_actionable_matching_reality — verdict-matches-reality via real Playwright clicks.

## risks
- Settle heuristic quiesces during the pre-insert delay or animation gap → miss the change. → Mitigated: quiet timer never fires before the first mutation; `maxWaitMs` ≫ delay+anim; animations awaited before geometry. Residual risk accepted for v0.1; hardening tracked as DW-01.
- Geometry read vs Playwright verdict disagree (the core 60% failure) → Mitigated: Playwright's trial verdict is authoritative and wins; disagreements are surfaced (`agreed:false`), tracked as DW-02.
- Token win is thin on a tiny controlled page → Accepted and stated plainly: v0.1 proves the *two missing dimensions* (geometry + actionability), not token savings; savings are a large-SPA property.
- Attribute-churn noise on surviving elements → Accepted for v0.1 (controlled cases don't touch surviving-element attributes); net-compare + semantic whitelist deferred to v0.5 noise tuning.
