# v0.8 flagship plan — offline `deltawright diagnose-trace <trace.zip>`

**Status:** RESEARCH DONE (feasibility recon 2026-07-11), BUILD NOT STARTED. Reserved for a **dedicated
run** — it is Size-L and defines v0.8, so it is deliberately kept out of the v0.7.x hardening line.

**Goal:** an offline CLI that reads a standard Playwright **trace.zip**, reconstructs a synthetic
delta, and runs the SAME `diagnose()` engine **offline** to explain a CI failure post-hoc — no re-run,
no live browser. "The agent reads your failing run." This is Wave-3 #9 of
[`v0.7-forward-roadmap.md`](v0.7-forward-roadmap.md), and it is **NOT** #25 (no HAR bench, no Claude
tokenizer — just reads a trace).

---

## Feasibility verdict: (b) TRACTABLE but genuinely Size-L / risky — not an afternoon.

The **diagnosis half is nearly free** (the engine + a proven reconstruction pattern already exist).
The **trace-reading half** rests on an **undocumented, internally-versioned Playwright format**
(currently trace **v8**, 8 historical migration passes) shipped only as **minified bundled code** — no
public type API. A useful, honest MVP is achievable by reading **only the action + error** (skip
DOM-snapshot reconstruction entirely). Full before/after DOM-diff + geometry is the large, fragile
part and should be **deferred**.

---

## What's reconstructable from a trace (and what is NOT)

`trace.zip` contains `trace.trace` (JSONL — one JSON event per line), `trace.network`, `trace.stacks`,
`resources/`. Key typed events (from `playwright-core/lib/coreBundle.js` event switch):
`context-options` (carries `version`, `browserName`, `playwrightVersion`), `before`→`action`, `input`,
`log` (`{callId,time,message}` — the actionability call-log), `after`
(`{callId, afterSnapshot, endTime, result, error, ...}`), `console`, `error`, `frame-snapshot`,
`resource-snapshot`. An **action** event carries `method` (click/fill/…), `params` (incl. **selector**),
`point`, `error`, and named `before/input/after` snapshot refs.

| `diagnose()` needs | Reconstructable? |
|---|---|
| `node.actionability.playwright.error` (PW error string) | **YES, verbatim** (`after.error` / `log`). The strong signal. |
| action method + selector | **YES** (`method` + `params`). For the human report. |
| `node.actionability.verdict` | **Synthesized** NOT-actionable for the failed target → suspected. |
| `node.geometry` (rects, hitSelf, coveredBy, offscreen, pointerEvents) | **NO** — snapshots store tag/attrs/text/viewport but **no per-node geometry**; recovering it needs re-rendering the HTML in a browser (breaks "no live browser"). |
| `geometryVerdict`, `agreed` | **NO** (no independent geometry) → `n/a`/synthesized. |
| `stats.*` (`hitMaxWait`, `droppedBackground`, `recurringInsert`, `lateStructural`, `detachedReRender`, `crossBoundarySkipped`, `injectionBlocked`) | **NONE** exist in a trace — all live-observer artifacts. Zero them. |

**Net:** only the **error-string-grounded** path (`codeFromPlaywrightError`, `diagnose.ts:50`) survives
offline. The geometry-grounded codes (`off-screen`, `covered-by-overlay`, `pointer-events-none`,
`not-visible`) and all stats/timing codes are **not** reconstructable from a trace. That is fine — the
error string is the strong, honest signal, and the taxonomy phrases PW emits (`intercepts pointer
events`, `not enabled`, `waiting for element to be visible`, …) are exactly what `codeFromPlaywrightError`
keys on.

## The reuse: `syntheticDelta` is already #9's core

`src/reporter/triage.ts` `syntheticDelta(errorMessage)` ALREADY builds a one-node `Delta` with
`geometry:null`, `actionability.verdict:'NOT-actionable'`, `playwright:{actionable:false,error}`,
`agreed:true`, and zeroed stats, then calls `diagnose(...)`. **#9 = trace.zip → extract `after.error` +
`method`/`params` → (reuse/clone) `syntheticDelta` → `diagnose()` → CLAMP to suspected →
`render(..., {diagnostics:true})`.** `render` (`serialize.ts:141`) + `summarizeDiagnoses`
(`host/summarize.ts`) are reusable for output.

---

## The one real trap: HONESTY CLAMP (DW-03)

The synthetic-delta path yields **`confirmed`** (an authoritative PW error → `assessConfidence({source:
'playwright'})` → confirmed; `triage.ts` accepts this because its error is from a LIVE run). #9 must
NOT — everything is RECONSTRUCTED, not live-probed. There is **no existing whole-delta downgrade flag**;
the honest mechanism is a **~5-line post-`diagnose()` clamp** over `DiagnosedDelta.diagnoses` capping
each `confidence` at `suspected` via `confidence.ts`'s exported `DOWNGRADE` / `CONFIDENCE_ORDER` /
`atLeastAsConfident`. Additive, pure, never touches the shared engine. **It is load-bearing — needs its
own test asserting NO emitted `confidence === 'confirmed'`.**

_Design nuance to decide:_ the trace error is the REAL PW error from the actual run — arguably as
authoritative as `triage.ts`'s (confirmed) path. Strictly, only reconstructed-DOM/geometry signals must
be suspected. **Recommendation: honor the roadmap — clamp EVERYTHING to `suspected`** (simplest, safest,
matches the "reconstructed, not live-probed" story) and document the nuance.

## Recommended MVP (honest; ships the flagship promise, dodges the fragile half)

1. **Unzip `trace.trace` dependency-FREE** — Node built-in `zlib` (`inflateRawSync`) + minimal ZIP
   central-directory parsing to extract the one entry. (Avoids a new runtime dep — the package has only
   4. If hand-rolled ZIP proves too fiddly against real fixtures, reconsider a tiny pure-JS lib like
   `fflate`, but that is an owner supply-chain decision.)
2. **Parse JSONL events**; assemble actions from `before`/`action` + `after` (match on `callId`); pick
   the **FAILED** action (`after.error` present / timeout / non-ok `result`).
3. **Build a `syntheticDelta`-style one-node `Delta`** from `error.message` (+ `log`), `geometry:null`,
   zeroed stats. (Reuse/clone `triage.ts`.)
4. `diagnose(delta)` → **clamp all confidences to `suspected`** → `render(..., {diagnostics:true})`, plus
   the action `method` + selector (`params`) in the header.
5. **Version-guard:** read `context-options.version`; if `> 8` (or `< min supported`) → refuse with a
   clear "unsupported trace version — regenerate with a supported Playwright" message. NEVER mis-parse.
6. **CLI:** add `case 'diagnose-trace':` to `src/cli.ts`'s flat `switch` (alongside `mcp`/`checksum`/
   `aggregate`); lazy-import the reader; print `render(...)`.

**This MVP needs NO DOM-snapshot parsing** — it dodges the hardest, most fragile part while delivering
"agent reads your failing run."

**Defer (the L-sized, risky half, a separate follow-up):** before/after DOM **snapshot reconstruction**
(resolve the `[[refIndex, nodeIndex]]` reference chain), structural before/after **diff** for multi-node
deltas, and any **geometry** (impossible offline without re-rendering). Also defer trying to recover
`off-screen`/`covered`/`pointer-events` codes — without rects they would be guesses.

## Risk surface (ranked)

1. **Format fragility (HIGH)** — internal, minified, 8 migrations; a PW upgrade can shift field
   names/version. Mitigate: parse the JSONL **yourself** (do NOT import `playwright-core` internals),
   pin behavior to trace `version`, hard-fail on unknown versions, and add fixture traces (real
   `trace.zip`s, PW is installed) as goldens per supported version.
2. **Honesty-clamp correctness (MEDIUM)** — the shared `diagnose()` yields `confirmed` for authoritative
   errors, so the clamp is load-bearing; dedicated test required.
3. **Snapshot fidelity (MEDIUM–HIGH, only if the deferred half is built)** — reference-chain resolution
   + no geometry make any snapshot-derived cause inherently lossy/suspected.
4. **New zip dependency (LOW)** — avoided by the dep-free approach; revisit only if hand-rolled parsing
   is too fragile.
5. **Test fixtures (MEDIUM)** — no trace fixtures in-repo; must generate real `trace.zip`s (a
   `ctx.tracing.start({snapshots:true}) → failing action → tracing.stop({path})` script) as golden
   inputs. **This is where much of the effort lives.**

**Effort:** the MVP (error-only, clamped, CLI-wired, version-guarded, fixtures + tests) is a focused
**multi-day** task — mostly fixtures, version-guarding, and the clamp test. The snapshot-diff enrichment
is a separate, larger follow-up and must NOT gate the flagship.
