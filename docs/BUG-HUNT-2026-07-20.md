# Deltawright — repo bug-hunt (2026-07-20)

A four-agent parallel sweep of `main` after the authoring-enhancer chapter shipped (pageMap +
observeEffectSettled + scoreSelectors + the north-star proof). Each finding was verified against the
code. **✅ FIXED** items were addressed in the same hardening pass that added this doc; the rest are the
**pick-next** backlog. Severity: HIGH = correctness/CI-reliability now; MED = real bug, bounded blast
radius; LOW = latency/polish/edge.

---

## A. Tests / build / packaging / CI

- **✅ FIXED [HIGH] Flaky timing-assertion family (5 sites).** The `framework-quiescence` flake (241ms
  under load) was one of five identical `settleMs < BUSY_MS-30` (~220ms ceiling, ~90ms headroom)
  assertions — also `framework-quiescence.spec.ts:30`, `quiescence.spec.ts:28`, `quiescence.spec.ts:79`,
  `wait.spec.ts:109`. The default settle (~130-160ms, `DEFAULT_SETTLE.quietMs=120`) is independent of
  the busy window, so **widening the busy/delay window 250→600ms** (fixtures + spec constants) enlarges
  the plain-vs-busy margin to ~320ms — robust under load. (Alternative considered: assert the *relative*
  invariant `plain.settleMs < q.settleMs`.)
- **✅ FIXED [MED] `release.yml` has no `concurrency:` group** — the `npm view … && skip` guard is
  TOCTOU-racy across concurrent runs (two tag pushes both pass "not published" → both publish → the
  loser fails red). Added `concurrency: { group: release-${{ github.ref }}, cancel-in-progress: false }`.
- **[LOW] `ci.yml` has no `concurrency:` group** → rapid PR pushes run redundant CI (cost only). Add
  `group: ci-${{ github.ref }}, cancel-in-progress: true`.
- **[LOW] Supply-chain hygiene:** actions pinned to floating major tags (`@v7`) not commit SHAs;
  `release.yml` runs unpinned `npm install -g npm@latest`. Worth SHA-pinning for an OIDC publisher.
- **[LOW] `bg-churn.spec.ts:24`** (`recurringInsert >= 4` in a 400ms cap) has margin but is theoretically
  load-sensitive. (The `effect-settle`/`northstar` `appearedMs >=` lower bounds get *safer* under load.)

### ESM/CJS dual build — VERDICT: **NEEDS-CARE** (feasible, not a drop-in) — DEFERRED (design decision)
`tsup format:['esm','cjs']` works **only with `shims:true`** and only if the bins stay ESM:
1. Without shims, `import.meta.url` is `undefined` in CJS → the top-level `const new URL('./injected/
   observer.global.js', import.meta.url)` in `inject.ts` throws at import time (`require('deltawright')`
   fails to load). Same for `cli.ts`. → `shims:true` mandatory.
2. tsup's shim uses `new URL('file:'+__filename)` (not `pathToFileURL`) → a path with spaces can
   mis-resolve the observer candidate. Patch the shim to `pathToFileURL`.
3. The "is main module" check (`mcp/server.ts`) compares encoders that differ under shims → dual-build
   only the 6 **library** entries; keep the `deltawright`/`deltawright-mcp` **bins ESM**.
4. Cross-format dynamic `import()` + the dual-package hazard are **low risk** (DW's singleton lives in
   `window.__deltawright`, not Node module state).
There is a working consumer workaround today: `await import('deltawright')` from CJS.

## B. Core delta / observe engine

- **✅ FIXED [MED] `live-routing.ts` — over-broad client-abort exclusion dropped a real backend fault.**
  `/ERR_ABORTED|aborted/i` also matched `net::ERR_CONNECTION_ABORTED` (a connection dropped by the
  network/server), silently excluding a genuine infra fault. Now `/\bERR_ABORTED\b/i` (only the client
  sentinel). Regression test added.
- **✅ FIXED [MED] `actAndObserve.ts` — child-frame late-watch MutationObserver leaked.** With
  `frames:true` + `lateWatchMs>0`, each child frame started a `lateObserver` never torn down (only the
  main page's `lateResult()` runs), and the frame's late signal is discarded anyway. Now child frames
  get `{ ...settle, lateWatchMs: 0 }`.
- **[MED] `observer.ts:~108` — synchronous XHR permanently leaks the in-flight counter.** For a sync
  `XMLHttpRequest`, `loadend` fires *during* `send()` (before `apply` returns), so the `{once:true}`
  listener is attached too late and never decrements → `isQuiescent()` stuck `false` for the page's life
  → every `awaitQuiescence` settle waits to the cap. Sync XHR is rare/deprecated, and the fix is
  intricate (increment+listen before apply, but then undo on a sync throw), so DEFERRED. **Fix:**
  attach+increment before `apply`, decrement in a `catch` on a synchronous throw.
- **[LOW] `actAndObserve.ts` — `awaitQuiescence` never armed inside child frames** (`enableQuiescence()`
  runs on the main page only). A child frame's `isQuiescent()` is always true → the frame silently skips
  the network gate. Fix per-frame or document as main-frame-only.
- **[LOW] `observer.ts` `settleAnimations` — infinite/paused animations burn the full `animMaxMs`.** A
  looping spinner's `.finished` never resolves → the timeout always fires. **Fix:** exclude
  infinite-iteration + `paused` animations from the awaited set. (Behavior change — validate against the
  animation tests.)
- **[LOW] `screenshot-diff.ts` — pixel-region rect is in device pixels on HiDPI** (`deviceScaleFactor>1`),
  inconsistent with CSS-pixel DOM geometry. Confined to the opt-in `screenshotFallback` empty-DOM path.

### Clean: actionability reconcile (DW-02/03), input-integrity classify, coalesce, checksum, live-routing
teardown/privacy, anchor-rescue, baseline lifecycle, shadow-root re-observation.

## C. New primitives (pageMap / effect-settle / scoreSelectors)

- **✅ FIXED [MED] `observeEffectSettled` leaked the baseline footprint.** The delta path clears
  `bg{Text,Attr,Insert,Remove}` in `collect()`, but `observeEffectSettled` cleans up with `reset()`,
  which never touched them → a later `{baseline:false}` call (contra its doc) ran against stale
  background. Now `waitForEffectSettle` consumes the footprint in a `finally` (symmetric with `collect`;
  NOT in `reset()`, which `arm()` calls after `sampleBaseline`). Regression test added.
- **✅ FIXED [LOW] `pageMap` reconcile flagged spurious `geomDisagreesWithPlaywright` on off-screen
  nodes** — a trial-click auto-scrolls an off-screen node into view and passes (fabricating a
  disagreement + mutating the scroll position `scan` already captured). Off-screen nodes now take the
  geometry path under reconcile (authoritatively NOT-actionable-here; no probe, no scroll). Regression
  test added.
- **[MED] A removal-dominated effect seeds the region from the removed node's PARENT.** Closing a modal
  appended to `<body>` → `m.target === body` → region ≈ whole page → PHASE 2 every mutation intersects →
  `hitMaxWait` on a live page (the global-quiescence failure R1 avoids). Detached nodes have no rect, so
  it's a genuine hard limit — honest-inconclusive (`hitMaxWait`, not a fake settle), but real. DEFERRED
  (needs a design decision: clamp region to viewport / capture last-known rects / report region=null +
  effectAppeared=true for top-level removals).
- **[LOW/MED] Multiple/nested overlays collapse to one `layer 1` and can carry the WRONG label** — a
  `role=alert` toast + a modal → a node in overlay B labeled with overlay A's name. Track the label per
  overlay root.
- **[LOW] Sticky/fixed page chrome is classified as an "overlay" layer** — a sticky header covering
  scrolled content reports as `layer 1 overlay`. Add a heuristic (full-width top-anchored sticky ≠ modal).
- **[LOW] `appearedMs` is stamped at drain time, not mutation time** (MutationRecords carry no
  timestamp) — soften the doc wording.

### Clean: `scoreSelectors` (synthesized geom candidate can never be `bestDurable`; escaping/round-trip
guarding/status ladder/concurrency sound), `isGeneratedId`/`isDynamicText`, `zoneOf`.

## D. Offline / reporting / MCP surfaces

**Notably well-defended** — the security-critical surfaces are clean (verified): aggregate HTML routes
every dynamic string through `escapeHtml` (no XSS breakout from a malicious side-car); reporter
`sanitize()` blocks path traversal; the zip parser validates offsets/bounds + refuses
ZIP64/encryption + tolerates truncation; confidence/taxonomy honesty invariants hold. Only:

- **✅ FIXED [LOW] `mcp/server.ts` — `serverInfo.version` hardcoded `'0.1.0'`** while the package is
  0.9.3. Now reads the real version from `package.json` (like `cli.ts`).
- **[LOW] `read-trace.ts:~85` — `harnessBucket` can mislabel a 4xx line as `5xx/gateway`** when the line
  carries an incidental 5xx-shaped number (`404 (took 503 ms)`). Routing decision unaffected (cosmetic
  label). Classify off the matched status token, not a free-floating `\b5\d\d\b`.
- **[LOW] `read-trace.ts:~228` — `capText` can split a surrogate pair** (slice by UTF-16 unit) → a lone
  `�`. Slice by code points.
